# v1.11-dev Compressed Summary

> Last updated: 2026-05-26. Full pre-compression slice notes are preserved in [v1.11_full_2026-05-26.md](v1.11_full_2026-05-26.md).

## Intent

v1.11 is the subsystem directory and non-Main data/algorithm boundary version. It should preserve behavior, public APIs, scene references, and Telemetry schema while making future content/config edits less dependent on reading large runtime scripts.

## Boundary Rules

- `Main.gd` remains the match-global state and orchestration owner.
- Tuning/catalog/data helpers own static values, ids, descriptors, resources, and fallback values.
- Formatters/builders/resolvers own text, UI construction, display specs, and visual lookup.
- Evaluators should be side-effect-light and receive explicit context.
- Controllers/directors/planners may own bounded domain process state, but not match-global state.
- Stores own persistence only.

## Retained Responsibilities

- `Main.gd`: scene wiring, match state, zone/mission/player/alive-count ownership, pressure trigger/effect flow, game-over flow, Telemetry hooks.
- `MissionTracker.gd`: active mission/pressure state, counters, hooks, public wrappers, pressure descriptor snapshots, and context assembly.
- `Player.gd`: movement, combat, heal, artifact, pickup, HUD update, zone warning, Sfx, and Telemetry runtime behavior.
- `Bot.gd`: AI state machine, perception, navigation, loot/supply decisions, combat, death/drop behavior, and Main-owned state reads.
- `Pickup.gd`: runtime nodes, focus/LOS, cluster-label behavior, collection side effects, Telemetry, and lifecycle.

## Current Risks

- `Bot.gd` remains large, but further extraction should wait for behavior coverage or a clearly isolated owner.
- Mission helpers are numerous but currently coherent; avoid more mission splitting unless new content exposes concrete drift.
- Code-built catalogs/tuning are acceptable first-pass boundaries. JSON/resource migration should wait for content-volume pressure.
- `ARCHITECTURE.md` and `IMPACT_MAP.md` remain large reference docs; open them only for structural/code-impact work.
