# v1.12-dev Summary

> Last updated: 2026-05-28.

## Intent

v1.12 adds Complex Artifacts after v1.10 Main cleanup and v1.11 subsystem/data-boundary closure. The goal is replay variation through small, bounded gameplay effects, not broad content expansion.

## Boundary Rules

- `ArtifactCatalog.gd` owns artifact ids, selection descriptors, display text, and catalog-owned numeric effect data.
- Player-owned artifact runtime helpers may own per-match trigger state for player artifacts.
- Player-owned artifact visual helpers may own player-attached nodes and event-driven presentation, but not gameplay decisions.
- `Player.gd` wires runtime helpers into existing movement/combat/damage/heal flows and applies returned player-state changes.
- `Main.gd` keeps selection/apply orchestration only.
- Telemetry additions must be explicit and must not change existing score schema.

## v1.12.1 Complex Artifacts Scope

- Chose **Emergency Shell** as the first implementation candidate.
- Rationale: one-shot low-HP shield has bounded state, visible feedback through existing shield HUD, and limited balance risk.
- Deferred candidates:
  - Glass Capsule: passive stat artifact; low boundary value as the first complex slice.
  - Ghost Grass: touches stealth/perception timing.
  - Pulse Scanner: needs directional HUD/minimap behavior.
  - Marked King: touches kill reward and exposure state.
  - Overheat Barrel: touches sustained-fire tuning and weapon heat behavior.
