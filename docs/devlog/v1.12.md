# v1.12-dev Summary

> Last updated: 2026-05-27.

## Intent

v1.12 adds Complex Artifacts after v1.10 Main cleanup and v1.11 subsystem/data-boundary closure. The goal is replay variation through small, bounded gameplay effects, not broad content expansion.

## Boundary Rules

- `ArtifactCatalog.gd` owns artifact ids, selection descriptors, display text, and catalog-owned numeric effect data.
- Player-owned artifact runtime helpers may own per-match trigger state for player artifacts.
- `Player.gd` wires runtime helpers into existing movement/combat/damage/heal flows and applies returned player-state changes.
- `Main.gd` keeps selection/apply orchestration only.
- Telemetry additions must be explicit and must not change existing score schema.

## v1.12.1 Complex Artifacts Scope

- Chose **Emergency Shell** as the first implementation candidate.
- Rationale: one-shot low-HP shield has bounded state, visible feedback through existing shield HUD, and limited balance risk.
- Deferred candidates:
  - Glass Capsule: passive stat artifact; low boundary value as the first complex slice.
  - Ghost Grass: touches stealth/perception timing.
  - Pulse Scanner: needs directional HUD/minimap behavior.
  - Marked King: touches kill reward and exposure state.
  - Overheat Barrel: touches sustained-fire tuning and weapon heat behavior.

## v1.12.2 Emergency Shell First Implementation

- Added Emergency Shell to `ArtifactCatalog.gd` with catalog-owned threshold and shield values.
- Added `PlayerArtifactRuntime.gd` for one-match trigger state and explicit health/shield trigger decisions.
- `Player.gd` now configures the runtime helper, applies returned Emergency Shell shield updates after non-lethal damage, and shows status flash feedback.
- `Telemetry.gd` now records selected artifact id and artifact event counts without changing score schema.
- Added `tools/verify_artifact_runtime.gd` smoke coverage for threshold, shield amount, and one-shot behavior.
- Verified with `git diff --check`, artifact runtime smoke, Godot headless quit, one normal simulation (75.0s/stage 3), and one Hell simulation (45.1s/stage 2).

## v1.12.3 Emergency Shell Readability Check

- Added `tools/verify_artifact_selection_layout.gd` to smoke-test artifact selection catalog text, Emergency Shell presence, card count, and default viewport row width.
- Verified five artifact cards fit the default 1280px viewport at 796px row width.
- Kept Emergency Shell values unchanged pending actual manual play/screenshot review.
- Shortlisted Ghost Grass as the next candidate because its bush-exit stealth grace is bounded player runtime state and has smaller UI/system blast radius than Pulse Scanner, Marked King, or Overheat Barrel.
- Verified with `git diff --check` and artifact selection layout smoke.

## Planned Follow-Ups

- Later v1.12: choose the next artifact only after Emergency Shell proves the runtime boundary in actual play.
