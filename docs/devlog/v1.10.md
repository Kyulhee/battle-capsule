# v1.10-dev Slice Summary

> Compact summary extracted from the full devlog. See [DEVLOG_full_2026-05-13.md](DEVLOG_full_2026-05-13.md) for exact historical entries.

## Status Clarification

- v1.10 is still in progress. The entries below are completed slices, not a completed Main slimdown release.
- `Main.gd` remains the orchestration root and still contains substantial UI, match setup, event, and state-wiring code.
- The v1.10 target is practical edit isolation: routine config, catalog, asset, and item-display work should avoid unrelated `Main.gd` sections.
- Before v1.11, finish or explicitly defer v1.10.x item/asset readability polish and complete at least one larger structural closure slice such as `MenuController` or `MatchBootstrap`.

## SupplyDropController Boundary

- Added `src/core/SupplyDropController.gd`.
- Moved supply telegraph timing, position roll, pillar progress, consumable cluster count, and cluster offsets out of `Main.gd`.
- `Main.gd` still owns supply state exposed to minimap and actual node creation.

## Regression Sentinel Summary

- Added regression sentinel reporting to `tools/analyze_results.py`.
- The current testing direction treats longer matches as acceptable unless combat/damage/perception loops collapse.

## Catalog/UI Data Boundaries

- `ArtifactCatalog`: starting artifact choice data and modifier definitions.
- `DifficultyCatalog`: difficulty UI labels, descriptions, and colors.

## v1.10.1 Display Patch

- Focused pickup floor marker was toned down by reducing radius, alpha, and emission while preserving focus selection logic.
- Runtime death drops now use `DropDisplayCatalog` for weapon, ammo, and heal display names.
- Player death drops no longer create English pickup labels such as `Pistol`, `Shotgun`, `Assault Rifle Ammo`, `Health Potion`, or `MedKit`.
- Bot death drops now share the same display catalog instead of owning duplicate naming helpers.

## v1.10.2 Icon Sizing Patch

- `tools/sync_generated_icons.ps1` now supports per-icon `VisualScale` overrides and `-OnlyCategory` filtering.
- Runtime weapon icons were re-synced with weapon-only filtering.
- Pistol/knife were scaled down, while AR/shotgun/railgun were slightly expanded to reduce the optical size mismatch caused by square generated canvases.
- HUD slot rendering and pickup decal code were not changed.

## Bot Objective Awareness Patch

- Loot/objective chase now separates navigation direction from view direction: bots continue moving to the pickup while scanning around that objective heading.
- Footstep and ambient awareness can set scan alerts during loot chase, so nearby movement can pull the bot's view without canceling the route.
- Opportunistic loot can be interrupted by a fully revealed enemy, but recovery/combat-loot runs keep their objective; direct damage and loud-gunshot overrides remain the higher-priority interruption paths.
- Telemetry hook names and JSON schema were not changed.
