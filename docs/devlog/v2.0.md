# v2.0-dev Summary

> Last updated: 2026-05-30.

## Intent

v2.0 prepares the game for larger maps and higher player counts by making map scale, match scale, spawn/loot density, zone profile, minimap/full-map metadata, and selected visual props explicit data before tuning toward 99 players.

## Boundary Rules

- `Main.gd` remains the match-global orchestrator and state owner.
- `MapDefinition.gd` owns definition data and validation, not scene mutation.
- `WorldBuilder.gd` continues to build world geometry from explicit spec data and must not trust imported GLB collision as gameplay authority.
- `Minimap.gd` and future Full Map UI render map/zone/marker data only; they do not own match logic.
- Scale increases must use validated presets before 99-player tuning.

## v2.0.1 MapDefinition Compatibility Loader

- Added `src/core/MapDefinition.gd` as a compatibility wrapper over current `MapSpec` JSON.
- Supports legacy MapSpec JSON directly and a wrapper format with `map_spec`, `match`, `runtime`, `zone`, and `scale_presets` sections.
- Exposes sanitized match, runtime, and zone tuning merged with `GameConfig`-compatible fallbacks.
- Adds validation for id/name, world size, POI bounds, obstacle bounds, spawn radius sanity, and loot-capable POI coverage.
- Added `tools/verify_map_definition.gd` to pin legacy wrapping and wrapper-format match overrides.

**Verification**

- `verify_map_definition.gd` passed: `mountain_forest_alpha`, 7 POIs, 35 obstacles, wrapper bot override 24.
- Godot headless project load passed with expected AssetCatalog warning only.
- One normal simulation passed.

## v2.0.2 Definition-Driven Scale Preset Merge

- Added baseline match/runtime/zone values and `baseline` / `medium_24` scale presets to `data/mapSpec_example.json`.
- `Main.gd` now loads `MapDefinition` before world generation and applies the selected scale preset into match tuning and runtime tuning.
- `MatchRuntimeTuning.gd` and `LootSpawner.gd` now support `hotspot_density_mult` for preset-driven POI loot density scaling.
- `MatchTuning.gd` accepts `scale_preset` / `map_scale_preset` command-line args for smoke scale runs.
- `tools/verify_map_definition.gd` verifies preset match overrides and runtime loot density overrides.
